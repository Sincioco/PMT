Executive Summary

PMT now supports a usable project delivery workflow from intake through sprint execution, QA, deployment, documentation, and scrum communication. The application has a modern dark theme, seeded demonstration data for PMT and LMS projects, login/password support, avatar display, and role-based administration features.

Recent work has improved stakeholder visibility through a cleaner Dashboard, click-through navigation between Dashboard, Sprints, and Tasks, Board controls for hiding empty columns, and read-only-first Documentation viewing. The result is a more practical tool for daily tracking and management review.

Current Capabilities

Dashboard

  • Project and sprint progress summaries.
  • Clean default view with details hidden until requested.
  • Show More Details per sprint and Show All Details toggle.
  • Click-through from sprint/task summaries to filtered views.

Kanban Board

  • Columns follow the full delivery workflow.
  • Drag-and-drop task status changes.
  • Quick create actions for tasks and bugs.
  • Hide Empty Columns and Show All Columns controls.

Tasks and Bugs

  • Project, sprint, status, assignee, completion, and age filters.
  • Parent/sub-task rendering with parent percent auto-calculation.
  • Bug Fix task type label and visual bug indicator.
  • Read-only task view with edit transition.

Projects and Sprints

  • Project-specific teams and sprint-specific member selection.
  • Sprint status graphs based on QA Passed completion.
  • Carry-forward logic for unfinished work and bugs.
  • Assignment lists limited to project/sprint participants.

Scrum

  • Daily scrum entries by project.
  • Newest entries shown first in table format.
  • Duplicate entry support with date defaulted to today.
  • Prompt placeholder for yesterday, today, and roadblocks.

Documentation

  • Renamed from Blogs to Documentation for business clarity.
  • Rich text documentation with seeded examples and images.
  • Existing documents open read-only first.
  • Hyperlinks open in a new browser tab.

Technical Foundation

  • Built with ASP.NET Core and .NET 6.
  • Uses ADO.NET and stored procedures, not Entity Framework.
  • Database objects are scoped under the pmt schema.
  • Seed scripts include users, avatars, projects, sprints, tasks, bugs, scrum entries, and documentation.
  • Default password is Password1, with user password change support.
  • Code has been kept intentionally simple and readable for junior developer maintenance.
