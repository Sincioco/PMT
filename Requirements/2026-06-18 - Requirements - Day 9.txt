Developer Task Tracking Visual Charts:
-------------------------------
1.) Create a graph button on the Dev Task View that is a toggle.  When it is enabled, the graphs/charts will appear in between the filter panel and the table of Dev Tasks.
2.) By default the chart view should be enabled (unless there is no data), but the user can toggle it off if they want to hide the charts and just see the table of Dev Tasks.
3.) Create Developer Task Tracking Charts that shows the number of Dev Tasks created per Sprint.
4.) Create another chart that shows and the number of Dev Tasks completed per Sprint.
5.) The charts should be interactive, allowing users to hover over data points to see the exact number of Dev Tasks created and completed for each Sprint.  This will provide more detailed insights into our development process and help us identify any potential bottlenecks or areas for improvement.
6.) Create a chart specific to the current sprint that shows the number of Dev Tasks created, completed, and still open.  This will help us understand the current sprint's development status and identify any potential issues that need to be addressed before the sprint ends.  This should be the first chart that appears when the user toggles on the chart view.
7.) If the user clicks on a chart and that data is associated to one specific element such as a Sprint or a Bug report or a Dev Task, popup a dialog box in read-only mode with details about that item.
8.) If there are other charts you find useful for tracking our development process, feel free to add them as well.  The goal is to provide as much insight and information as possible to help us improve our development process and ultimately deliver a better product to our users.
9.) The chart should not use any charting library.  It should be built from scratch using HTML, CSS, and JavaScript.  This will allow for greater customization and flexibility in the design and functionality of the chart.  Create a re-usable charting module that can be used for other types of charts in the future and to make it easier to maintain and update the charts as needed.  The charting module should be designed to handle different types of data and be easily customizable to fit the specific needs of our development process tracking.
10.) Be sure the chart has smooth animation.
